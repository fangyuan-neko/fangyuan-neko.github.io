hexo写作

写作流程

  1. 打草稿

    hexo new draft <name>

  2. 预览

    草稿默认不会显示在页面中，您可执行hexo s --draft，或是在_config.yml中把render_drafts参数设为true来预览草稿。

注: 如果要发布的话必须再渲染一遍，不然会把草稿一起发出去。

  1. 发表

    hexo publish [layout] <title>

  2. 推送

    hexo deploy

插图

插图是一个非常令人头疼的问题，可以参见如下方法

详情可先参见官网文档

在_config.yml中设置

post_asset_folder: true

然后给图片取一个简单的名字，如1.jpg，然后用{% asset_img 图片名 注释 %}的格式在你想插入图片的地方试一试吧

省略

在作者用的Ayer主题中，文章中插入<!--more-->就可以省略。
